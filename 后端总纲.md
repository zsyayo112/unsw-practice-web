## 数据库 Schema

### courses 表
```sql
CREATE TABLE courses (
  id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  code        text NOT NULL UNIQUE,   -- 'COMP9021'
  name        text NOT NULL,          -- 'Principles of Programming'
  description text,
  is_active   boolean DEFAULT true,
  created_at  timestamptz DEFAULT now()
);
```

### problems 表
```sql
CREATE TABLE problems (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  course_id       uuid REFERENCES courses(id),
  slug            text NOT NULL UNIQUE,     -- 'fibonacci'
  title           text NOT NULL,            -- 'Fibonacci Sequence'
  difficulty      text CHECK (difficulty IN ('easy','medium','hard')),
  topics          text[] DEFAULT '{}',      -- ['recursion']
  description     text NOT NULL,            -- Markdown
  starter_code    text NOT NULL,            -- Python 初始代码
  solution_code   text,                     -- 答案（对学生隐藏）
  hints           text[] DEFAULT '{}',
  order_index     integer DEFAULT 0,
  acceptance_rate numeric DEFAULT 0,
  is_published    boolean DEFAULT false,
  created_at      timestamptz DEFAULT now(),
  updated_at      timestamptz DEFAULT now()
);
```

### test_cases 表
```sql
CREATE TABLE test_cases (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  problem_id      uuid REFERENCES problems(id) ON DELETE CASCADE,
  input           text NOT NULL,            -- 'print(fib(0))'
  expected_output text NOT NULL,            -- '0'
  is_hidden       boolean DEFAULT false,
  order_index     integer DEFAULT 0
);
```

### user_progress 表（阶段二启用）
```sql
CREATE TABLE user_progress (
  id                uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id           uuid,                   -- 预留，阶段二关联 auth.users
  problem_id        uuid REFERENCES problems(id),
  status            text CHECK (status IN ('attempted','solved')),
  solve_count       integer DEFAULT 0,
  last_submitted_at timestamptz,
  UNIQUE(user_id, problem_id)
);
```

## API 接口设计

### GET /api/v1/problems
返回题目列表（不含 solution_code）

请求参数（可选）：
- `course` — 按课程筛选，e.g. `?course=COMP9021`
- `difficulty` — 按难度筛选
- `published` — 默认只返回已发布题目

响应：
```json
{
  "data": [
    {
      "id": "uuid",
      "slug": "fibonacci",
      "title": "Fibonacci Sequence",
      "difficulty": "easy",
      "topics": ["recursion"],
      "description": "## 题目\n...",
      "starterCode": "def fib(n):\n    pass",
      "hints": ["考虑递归的终止条件"],
      "orderIndex": 1,
      "acceptanceRate": 72.5,
      "courseId": "uuid",
      "testCases": [
        { "id": "uuid", "input": "print(fib(0))", "expectedOutput": "0", "isHidden": false }
      ]
    }
  ],
  "error": null
}
```

### GET /api/v1/problems/:slug
返回单题详情

### POST /api/v1/submit
提交代码（调 Judge0 CE 执行）

请求体：
```json
{
  "problemId": "uuid",
  "code": "def fib(n):\n    ...",
  "language": "python"
}
```

响应：
```json
{
  "data": {
    "status": "accepted",
    "testResults": [
      {
        "passed": true,
        "input": "print(fib(0))",
        "expectedOutput": "0",
        "actualOutput": "0",
        "error": null
      }
    ],
    "runtimeMs": 120,
    "message": "3 / 3 Tests Passed"
  },
  "error": null
}
```

### GET /api/v1/progress（阶段二，需要 token）
返回用户做题进度

## .py 文件解析脚本

处理现有的 COMP9021 题目文件，自动提取：
- 函数签名 → starter_code
- doctest 用例 → test_cases（input + expected_output）
- 文件名 → slug

```python
# scripts/import_problems.py 使用方式：
python import_problems.py --dir ./py_files --course COMP9021 --dry-run
python import_problems.py --dir ./py_files --course COMP9021 --import
```

解析规则：
- 函数体替换为 `pass` → starter_code
- `>>> f(...)` 行 → input
- 紧随其后的非 `>>>` 行 → expected_output（支持多行输出）
- 文件名去掉 .py → slug（需手动补充 title 和 description）

```markdown
# UNSW Practice Web Backend

## 项目背景
UNSW CS 学生学习平台的后端服务 + 管理后台。
配套 VSCode 插件：unsw-practice-vscode（已完成 MVP）
商业目标：为补课机构引流，平台本身免费。

## 当前阶段
Week 5：后端 API + Supabase 数据库 + 管理后台 + 题库导入

## 技术约束
- API 必须用 /api/v1/ 前缀（插件已按此调用）
- 响应格式统一：{ data, error }
- solution_code 字段不能出现在 API 响应里
- 管理后台用 ADMIN_SECRET 简单保护（阶段二再做完整 auth）

## 构建命令
- npm run dev    → localhost:3000
- npm run build  → 生产构建
- npm run lint   → ESLint

## 数据库
Supabase PostgreSQL
表：courses / problems / test_cases / user_progress
连接：通过 @supabase/ssr 包

## 代码规范
- TypeScript strict 模式
- API Route 必须处理 try/catch，返回 { error } 而不是抛异常
- 不在 API 响应里暴露 solution_code
```
